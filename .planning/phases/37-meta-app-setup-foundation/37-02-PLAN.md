    Create the foundation files for the meta module:

    **constants.ts** — Two exports only:
    - `META_GRAPH_API_VERSION = 'v22.0'` (string literal, pinned)
    - `META_BASE_URL = 'https://graph.facebook.com/v22.0'` (template literal using the version constant)
    No other imports. Zero dependencies (prevents circular deps per project pattern).

    **types.ts** — Contains:
    1. `MetaApiError` interface for raw Meta error response shape: `{ error?: { message?: string; code?: number; error_subcode?: number; fbtrace_id?: string; type?: string } }`
    2. `MetaGraphApiError` class extending Error with:
       - Constructor params: message, code?, errorSubcode?, httpStatus?, fbtraceId?
       - `name = 'MetaGraphApiError'`
       - Getter `isAuthError`: httpStatus===401 || code===190
       - Getter `isRateLimitError`: httpStatus===429 || code===4 || code===80007
       - Getter `isPermissionError`: code===10 || code===200
    3. `MetaCredentials` interface: accessToken, wabaId, phoneNumberId, phoneNumber, pageId, igAccountId, businessId, workspaceId (all string, nullable where appropriate)
    4. `MetaChannel` type: `'whatsapp' | 'facebook' | 'instagram'`

    **token.ts** — AES-256-GCM encryption per RESEARCH.md Pattern 2:
    - Import `createCipheriv`, `createDecipheriv`, `randomBytes` from `crypto`
    - Constants: ALGORITHM='aes-256-gcm', IV_LENGTH=12, AUTH_TAG_LENGTH=16
    - `getEncryptionKey()`: reads META_TOKEN_ENCRYPTION_KEY from env, decodes base64 to Buffer, throws if missing. Validate decoded length is exactly 32 bytes.
    - `encryptToken(token: string): string`: randomBytes(12) IV, cipher, pack as base64(iv + authTag + ciphertext)
    - `decryptToken(packed: string): string`: unpack base64, extract iv/authTag/ciphertext by byte offsets, decipher
    - IMPORTANT: Each encrypt call MUST use randomBytes(12) for IV. Never deterministic IV.
  </action>

    - constants.ts has zero imports
    - token.ts uses randomBytes(12) in encryptToken
    - MetaGraphApiError has all three getter methods
  </verify>

  <action>
    **api.ts** — Typed Graph API client per RESEARCH.md Pattern 1:
    - Import META_BASE_URL from ./constants, MetaGraphApiError from ./types
    - `metaRequest<T>(accessToken, endpoint, options?): Promise<T>`:
      - Builds URL as `${META_BASE_URL}${endpoint}` (endpoint starts with `/`)
      - Sets Authorization Bearer header + Content-Type application/json
      - Spreads options.headers after defaults (caller can override Content-Type for multipart)
      - On !response.ok: parse error body, throw MetaGraphApiError with code, subcode, httpStatus, fbtraceId
      - On ok: return data as T
    - `sendWhatsAppText(accessToken, phoneNumberId, to, text)`: POST to `/${phoneNumberId}/messages` with messaging_product=whatsapp, type=text
    - `sendWhatsAppTemplate(accessToken, phoneNumberId, to, templateName, languageCode='es', components?)`: POST to `/${phoneNumberId}/messages` with type=template
    - `verifyToken(accessToken, wabaId): Promise<boolean>`: GET `/${wabaId}?fields=id,name`, return true/false (try/catch)

    **credentials.ts** — Per RESEARCH.md Pattern 3:
    - Import createAdminClient from @/lib/supabase/admin
    - Import decryptToken from ./token
    - Import MetaCredentials, MetaChannel from ./types
    - `resolveByPhoneNumberId(phoneNumberId: string): Promise<MetaCredentials | null>`:
      - Query workspace_meta_accounts WHERE phone_number_id=X AND is_active=true, .single()
      - If no data, return null
      - Return credentials with decryptToken(data.access_token_encrypted) as accessToken
    - `resolveByPageId(pageId: string): Promise<MetaCredentials | null>`: Same pattern for page_id
    - `resolveByIgAccountId(igAccountId: string): Promise<MetaCredentials | null>`: Same pattern for ig_account_id
    - `resolveByWorkspace(workspaceId: string, channel: MetaChannel): Promise<MetaCredentials | null>`:
      - Query WHERE workspace_id=X AND channel=Y AND is_active=true, .single()
      - Decrypt and return

    All credential functions use createAdminClient() (bypass RLS per project pattern).
    Map DB row fields (snake_case) to MetaCredentials interface (camelCase) explicitly.
  </action>
  <verify>

    - credentials.ts imports createAdminClient from @/lib/supabase/admin
    - credentials.ts calls decryptToken on access_token_encrypted
    - All four resolve functions exist in credentials.ts
  </verify>
