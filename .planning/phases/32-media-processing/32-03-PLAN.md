**agent-production.ts -- Major restructure of whatsappAgentProcessor:**

The current function has a single `step.run('process-message')` that calls `processMessageWithAgent`. The new flow adds a media gate step BEFORE that:

```
Step 1: 'media-gate' -> processMediaGate(input)
Step 2: Based on result:
  - passthrough -> step.run('process-message') with transformed text (existing flow)
  - handoff -> step.run('execute-media-handoff') -> send "Regalame 1 min" + notify host + cancel timer
  - notify_host -> step.run('notify-host') -> create notification via domain layer (no handoff)
  - ignore -> return immediately (no processing)
```

Implementation:

1. Add `step.run('media-gate', ...)` as the FIRST step:
   ```typescript
   const gateResult = await step.run('media-gate', async () => {
     const { processMediaGate } = await import('@/lib/agents/media')
     return processMediaGate({
       messageType: event.data.messageType ?? 'text',
       messageContent: event.data.messageContent,
       mediaUrl: event.data.mediaUrl ?? null,
       mediaMimeType: event.data.mediaMimeType ?? null,
       workspaceId: event.data.workspaceId,
       conversationId: event.data.conversationId,
       phone: event.data.phone,
     })
   })
   ```

2. After the media gate, switch on `gateResult.action`:

   **Case 'passthrough':**
   Continue with existing flow. Replace `messageContent` with `gateResult.text` in the processMessageWithAgent call. This handles both normal text AND transcribed audio.

   **Case 'handoff':**
   Execute handoff workflow:
   ```typescript
   await step.run('execute-media-handoff', async () => {
     const { getWorkspaceAgentConfig } = await import('@/lib/agents/production/agent-config')
     const { executeHandoff } = await import('@/lib/agents/production/handoff-handler')
     const config = await getWorkspaceAgentConfig(workspaceId)
     await executeHandoff(conversationId, workspaceId, {
       handoffMessage: config?.handoff_message ?? 'Regalame 1 min, ya te comunico con un asesor',
     })
   })
   ```

   Also cancel any active silence timer by emitting agent/customer.message.

   WHY this step is needed: For 'handoff' (image/video/failed transcription), the UnifiedEngine
   is NOT invoked -- we skip directly to executeHandoff. Normally, the engine emits
   `agent/customer.message` in its step 6 (after intent detection), which cancels the silence
   timer. Since the engine is bypassed for media handoff, the timer would never be cancelled,
   causing a stale retake message to fire after the human agent has already taken over.
   For 'passthrough' (text, transcribed audio, recognized sticker, mapped reaction), the engine
   IS invoked and emits the event naturally -- no extra cancellation needed.
   For 'ignore' and 'notify_host', the engine is NOT invoked, but the timer SHOULD continue
   (bot stays active, customer didn't do anything significant).

   ```typescript
   await step.run('cancel-silence-timer', async () => {
     // Media handoff bypasses the UnifiedEngine, which normally emits
     // agent/customer.message (step 6) to cancel the silence timer.
     // Without this explicit cancellation, the timer would fire a stale
     // retake message after the human agent has already taken over.
     const { inngest: inngestClient } = await import('@/inngest/client')
     const { createAdminClient } = await import('@/lib/supabase/admin')
     const supabase = createAdminClient()
     const { data: session } = await supabase
       .from('agent_sessions')
       .select('id')
       .eq('conversation_id', conversationId)
       .eq('workspace_id', workspaceId)
       .eq('is_active', true)
       .order('created_at', { ascending: false })
       .limit(1)
       .single()

     if (session) {
       await inngestClient.send({
         name: 'agent/customer.message',
         data: {
           sessionId: session.id,
           conversationId,
           messageId: event.data.messageId,
           content: gateResult.reason,
         },
       })
     }
   })
   ```

   Return early: `return { success: true, newMode: 'handoff', mediaType: event.data.messageType }`

   **Case 'notify_host':**
   Notify the host without performing handoff (bot stays active). Use the domain layer
   `createTask` function (CLAUDE.md Rule 3: ALL mutations go through src/lib/domain/):
   ```typescript
   await step.run('notify-host-media', async () => {
     const { createAdminClient } = await import('@/lib/supabase/admin')
     const supabase = createAdminClient()
     const { createTask } = await import('@/lib/domain/tasks')

     // Fetch contact info for the notification description
     const { data: conv } = await supabase
       .from('conversations')
       .select('contact_id, profile_name, phone')
       .eq('id', conversationId)
       .single()

     const contactName = conv?.profile_name ?? conv?.phone ?? phone

     // Use domain layer for task creation (Rule 3: no raw inserts)
     await createTask(
       { workspaceId, source: 'inngest' },
       {
         title: `Notificacion: ${gateResult.reason}`,
         description: `Conversacion con ${contactName}. El bot sigue activo.`,
         priority: 'medium',
         status: 'pending',
         conversationId,
         contactId: conv?.contact_id ?? undefined,
       }
     )
   })
   ```
   Return early: `return { success: true, mediaType: event.data.messageType }`

   **Case 'ignore':**
   Log and return immediately:
   ```typescript

   return { success: true, ignored: true, mediaType: event.data.messageType }
   ```
