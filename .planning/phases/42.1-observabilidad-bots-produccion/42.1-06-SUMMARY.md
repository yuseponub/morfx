```diff
- import Anthropic from '@anthropic-ai/sdk'
```

No fallback models, no secondary Claude calls — each file has exactly one
`messages.create(...)` invocation, so exactly one `runWithPurpose` wrap per file.

## Out of Scope — Cross-Subsystem `new Anthropic(` Sites

Three `new Anthropic(` constructions remain outside `src/lib/agents/`:

| File | Purpose | Why out of scope for 42.1 |
|---|---|---|
| `src/lib/pdf/normalize-order-data.ts:238` | Extract structured order data from order-confirmation PDFs (Phase 28 robot) | Not an agent turn — batch PDF normalization pipeline. No customer-facing conversational flow. |
| `src/lib/ocr/extract-guide-data.ts:54` | OCR extraction from carrier guide images | Batch logistics pipeline, not a conversational turn. |
| `src/lib/domain/carrier-coverage.ts:343` | Classify shipping address → carrier coverage tier | Domain helper called from admin tooling, not from the agent turn hot path. |

**Phase 42.1's declared scope** (per CONTEXT.md and the 10-site inventory in
RESEARCH.md) is conversational-agent observability: capturing the AI calls and
pipeline decisions that drive customer-facing WhatsApp turns. The three files
above belong to separate subsystems (PDF normalization, OCR, logistics).
Instrumenting them would require separate handler wraps (each runs in its own
Inngest function or server action, none of which are wrapped in
`runWithCollector`) and their output would flow into a different observability
taxonomy.

**Recommendation:** if/when cross-subsystem AI cost tracking becomes a product
requirement, spin up a dedicated standalone plan to (a) migrate these three to
`createInstrumentedAnthropic`, (b) wrap their host handlers in
`runWithCollector`, and (c) extend the `AgentId` union to include a non-agent
`'system'` or `'batch'` tag. Plan 11 (gap audit) should log this as a known
boundary. **Do NOT tighten the lint invariant to `src/` globally until that plan
ships** — doing so would block the three non-agent constructions without a
migration path.
