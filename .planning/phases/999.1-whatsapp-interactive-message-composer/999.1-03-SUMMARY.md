# Phase 999.1 Plan 03: Server action sendInteractiveMessage Summary

Adds the `sendInteractiveMessage` server action to `src/app/actions/messages.ts`, mirroring `sendMessage` EXACTLY: `getRequestAuth()` auth gate → load conversation scoped by `workspace_id` → 24h-window re-check (defense-in-depth behind the composer UI gate D-02) → resolve `apiKey` from `workspaces.settings` → build `DomainContext` → delegate to `domainSendInteractiveMessage` → `revalidatePath('/whatsapp')`. This is the adapter-layer chokepoint the composer modal (Plan 04) will call. Per Regla 3 the action NEVER reads or branches on the WhatsApp provider — the domain (Plan 02) owns that single decision. A channel guard rejects non-WhatsApp conversations (FB/IG out of scope). All 12 existing tests stay GREEN (Regla 6).

## What Was Built

### Task 1 — `sendInteractiveMessage` server action mirroring `sendMessage` — commit `4bbfabe0`
- **(a) Import:** extended the existing domain import block with `sendInteractiveMessage as domainSendInteractiveMessage` from `@/lib/domain/messages`.
- **(b) Action** (`sendInteractiveMessage(conversationId, payload): Promise<ActionResult<{ messageId: string }>>`), copied verbatim from the `sendMessage` skeleton and adapted:
  1. **AUTH:** `const auth = await getRequestAuth(); if (!auth) return { error: 'No autenticado' }`; `workspaceId = auth.workspaceId` (T-999.1-V2 mitigated).
  2. **LOAD CONVERSATION:** `.from('conversations').select('id, phone, last_customer_message_at, status, channel').eq('id', conversationId).eq('workspace_id', workspaceId).single()`; not-found → `{ error: 'Conversacion no encontrada' }` (T-999.1-V4 — workspaceId from `auth`, never the client payload).
  3. **CHANNEL GUARD:** `if (conversation.channel !== 'whatsapp') return { error: 'Interactivos solo disponibles en WhatsApp' }` (FB/IG out of scope — RESEARCH Pattern 2).
  4. **24h WINDOW (verbatim from `sendMessage`):** missing `last_customer_message_at` OR `differenceInHours(...) >= 24` → `{ error: 'Ventana de 24h cerrada. Usa un template.' }` (T-999.1-D2 — defense-in-depth behind D-02; interactive is a session message).
  5. **RESOLVE apiKey:** `workspaces.select('settings')` → `apiKey = settings?.whatsapp_api_key || process.env.WHATSAPP_API_KEY`; missing → `{ error: 'API key de WhatsApp no configurada' }`. The meta_direct arm ignores `apiKey`; the domain resolves Meta creds itself. **The action never reads `whatsapp_provider`** (Regla 3 / T-999.1-D3).
  6. **DELEGATE + revalidate:** `ctx: DomainContext = { workspaceId, source: 'server-action' }`; `domainSendInteractiveMessage(ctx, { conversationId, contactPhone: conversation.phone, apiKey, ...payload })`; on failure → `{ error: result.error || 'Error al enviar mensaje interactivo' }`; unarchive if `status === 'archived'`; `revalidatePath('/whatsapp')`; return `{ success: true, data: { messageId: result.data!.messageId } }`.
