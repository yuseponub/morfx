# Phase 999.1 Plan 02: Domain sendInteractiveMessage chokepoint Summary

Adds the single domain chokepoint `sendInteractiveMessage(ctx, params)` to `src/lib/domain/messages.ts` — the one function that reads `whatsapp_provider` ONCE and branches `meta_direct` vs `360dialog` (Regla 3 / Phase 39), modeled verbatim on `sendTemplateMessage`. Provider decision lives ONLY in the domain (call-sites never decide — D-03). It stores the full interactive structure as JSONB (D-04) for the rich outbound bubble and adds only a CALLER for the legacy 360dialog `sendButtonMessage` (zero change to that function — Regla 6). The RED contract test from Plan 01 (`messages-interactive-provider.test.ts`) now turns GREEN, and the 4 existing tests stay GREEN.

## What Was Built

### Task 1 — import send360Buttons + SendInteractiveMessageParams interface — commit `146665d8`
- Extended the existing 360dialog import block in `messages.ts` by adding `sendButtonMessage as send360Buttons` (the new caller; `whatsapp/api.ts` itself untouched — Regla 6).
- Added `SendInteractiveMessageParams` modeled after `SendTemplateMessageParams`: union-discriminated on `interactiveType: 'buttons' | 'list'`, with `conversationId`, `contactPhone`, `apiKey` (360dialog key; meta arm ignores it), `body`, optional `header`/`footer`, and the per-type optionals `buttons` / `buttonLabel` / `sections`.
- Reused the existing `SendMessageResult` (L125) as the return data type — no new type invented.

### Task 2 — implement sendInteractiveMessage (provider chokepoint + JSONB store) — commit `b613d0e9`
- Added `export async function sendInteractiveMessage(ctx, params): Promise<DomainResult<SendMessageResult>>` after `sendTemplateMessage`, copying its structure verbatim.
- **Single provider read:** `readWhatsappProvider(supabase, ctx.workspaceId)` — exactly ONCE.
- **meta_direct arm:** resolves creds via `resolveByWorkspace(ctx.workspaceId, 'whatsapp')` (T-39-02 — never from input); missing creds → `{ success:false, error:'Credenciales Meta no configuradas' }`; buttons → `metaWhatsappSender.sendButtons(metaCreds, ...)`; list → `metaWhatsappSender.sendList(metaCreds, ..., buttonLabel ?? '', sections ?? [], ...)`; `wamid = resp.externalMessageId`.
- **360dialog arm (legacy, byte-identical):** list → returns `{ success:false, error:'lista no soportada en 360dialog (legacy)' }` BEFORE any send (D-03); buttons → `send360Buttons(params.apiKey, ...)`; `wamid = resp.messages?.[0]?.id`.
- Missing wamid → `{ success:false, error:'No se recibio ID de mensaje de WhatsApp' }`.
- **JSONB store (D-04):** builds `content` = `{ interactiveType, body, header?, footer?, buttons?, buttonLabel?, sections? }` (only present fields) and inserts into `messages` with `type:'interactive'`, `direction:'outbound'`, `status:'sent'`.
- insertError guard + conversations update (`last_message_at`, `last_message_preview:'[Interactivo]'`, unarchive `status:'active'`) — verbatim from `sendTemplateMessage`.
- catch block verbatim with log prefix `[domain/messages] sendInteractiveMessage failed:`.
