Create a module that generates minifrases for human-typed and AI-generated outbound messages using Haiku.

Follow the exact pattern from `message-classifier.ts` for Anthropic SDK direct calls:
- Import `Anthropic` from `@anthropic-ai/sdk`
- Create client with `process.env.ANTHROPIC_API_KEY`
- Use `claude-sonnet-4-20250514` (same model as MessageClassifier — Haiku not yet available per claude-client.ts MODEL_MAP)
- Use `max_tokens: 128` (minifrases are very short)

**The prompt** (MINIFRASE_PROMPT system prompt):
```
Genera una minifrase tematica (max 15 palabras) que capture la ESENCIA de este mensaje de WhatsApp.
La minifrase debe capturar los TEMAS cubiertos, no las palabras exactas.

Ejemplos:
"Veras los resultados desde los primeros 3-7 dias" -> "resultados en 3-7 dias, melatonina regula ciclo, magnesio relaja"
"El pago lo haces cuando recibes el producto en efectivo" -> "pago contraentrega en efectivo al recibir"
"Tranquilo, veras cambios desde la primera semana. No es un somnifero." -> "efectividad rapida, no es somnifero, regula ciclo natural"

Responde SOLO con la minifrase, sin explicaciones ni comillas.
```

**Export function:**
```typescript
/**
 * Generate minifrases for outbound messages that don't have one yet.
 * Mutates the entries array in-place, filling empty tema fields.
 * Uses a single batched Haiku call per message that needs a minifrase.
 *
 * @param entries - OutboundEntry[] where tema='' needs generation
 */
export async function generateMinifrases(entries: OutboundEntry[]): Promise<void>
```

Implementation:
1. Filter entries where `tema === ''` and `fullContent` exists (or content is available).
2. For each entry needing a minifrase, call Haiku to generate one.
3. Use `Promise.all()` for parallel generation (max ~5 concurrent since there are usually few human/AI messages per conversation).
4. Set `entry.tema = generatedMinifrase` in-place.
5. If Haiku call fails for an entry, set tema to a fallback: first 15 words of the content (degraded but functional).
6. Log each generation with createModuleLogger('minifrase-generator').

IMPORTANT: The function needs the full content of messages to generate minifrases. The outbound-registry.ts from Plan 01 stores content in `fullContent` for human/AI entries. If fullContent is missing, skip that entry (log a warning).

Handle edge cases:
- Empty content -> skip
- Haiku returns empty string -> use fallback (first 15 words)
- API error -> log and use fallback
- No entries needing generation -> return immediately

  <name>Task 2: NoRepetitionFilter class with 3 escalating levels</name>
  <files>src/lib/agents/somnio/no-repetition-filter.ts</files>
  <action>
Create the core filter class that implements the 3-level no-repetition check.

**Class structure:**
```typescript
import Anthropic from '@anthropic-ai/sdk'
import { createModuleLogger } from '@/lib/audit/logger'
import type { PrioritizedTemplate } from './block-composer'
import type { OutboundEntry, NoRepFilterResult, NoRepLevel2Decision } from './no-repetition-types'

const logger = createModuleLogger('no-repetition-filter')

export class NoRepetitionFilter {
  private client: Anthropic
  private minifraseCache: Map<string, string> = new Map()

  constructor(private workspaceId?: string) {
    this.client = new Anthropic({
      apiKey: process.env.ANTHROPIC_API_KEY,
    })
  }

  /**
   * Filter a block of templates through the 3-level no-repetition system.
   * Each template is checked sequentially through escalating levels.
   * Short-circuits at the earliest level that gives a definitive answer.
   *
   * @param block - Templates to filter (from BlockComposer output)
   * @param outboundRegistry - All previously sent messages with minifrases
   * @param templatesEnviados - Template IDs already sent (for Level 1)
   * @returns NoRepFilterResult with surviving and filtered templates
   */
  async filterBlock(
    block: PrioritizedTemplate[],
    outboundRegistry: OutboundEntry[],
    templatesEnviados: string[]
  ): Promise<NoRepFilterResult>
}
```

**Level 1 (ID lookup)** — inside filterBlock:
- For each template in block, check if `templatesEnviados.includes(template.templateId)`
- If yes: add to filtered with `{ template, level: 1, reason: 'Template ID already sent' }`
- Cost: 0ms, $0. Expected to catch ~60% of cases.

**Level 2 (minifrase Haiku comparison)** — private method `checkLevel2`:
- Only for templates that pass Level 1.
- Need the candidate template's minifrase: query from outbound registry (for templates, their minifrase is in the registry as 'plantilla' entries) OR query agent_templates directly.
- Actually, the candidate template is from the CURRENT block (not yet sent), so we need its minifrase from agent_templates. Add a helper that fetches minifrase for a template ID.
- Call Haiku with the LEVEL2_PROMPT:

```
Eres un detector de repeticion para un bot de ventas de WhatsApp.

Te doy la MINIFRASE de una plantilla que el bot quiere enviar, y las MINIFRASES de todo lo que ya se envio en esta conversacion (por bot, humano, o IA).

Tu tarea: determinar si la plantilla agrega informacion NUEVA o si ya fue cubierta.

Responde SOLO con un JSON:
{
  "decision": "ENVIAR" | "NO_ENVIAR" | "PARCIAL",
  "razon": "breve explicacion"
}

Reglas:
- ENVIAR: La plantilla tiene informacion que NO aparece en ningun mensaje previo
- NO_ENVIAR: La informacion de la plantilla ya fue cubierta completamente
- PARCIAL: Parte de la informacion fue cubierta, parte es nueva (necesita check mas profundo)
- Compara TEMAS, no palabras exactas. "precio 77900 con envio gratis" y "el costo es 77900 envio incluido" son el MISMO tema.
- Si la plantilla agrega un ANGULO DIFERENTE del mismo tema (ej: plantilla habla de severidad del insomnio, previo habla de efectividad general), es PARCIAL.
```

User message format:
```
MINIFRASE DE LA PLANTILLA CANDIDATA:
{candidateMinifrase}

MINIFRASES DE MENSAJES PREVIOS:
{outboundRegistry.map(e => `- [${e.tipo}] ${e.tema}`).join('\n')}
```

- Use `claude-sonnet-4-20250514` (same as MessageClassifier)
- `max_tokens: 256`
- Parse JSON response with jsonMatch pattern from message-classifier.ts
- Return 'ENVIAR' | 'NO_ENVIAR' | 'PARCIAL'
- On parse failure, default to 'ENVIAR' (fail-open: send rather than wrongly block)

**Level 3 (full context check)** — private method `checkLevel3`:
- Only for templates where Level 2 returned 'PARCIAL'.
- Fetch the full content of the candidate template (from agent_templates or from the block's .content field — it's already in PrioritizedTemplate.content!).
- Also need full content of the overlapping outbound entries. Use entries where tema matches. The outbound registry should have fullContent for human/AI messages. For template entries, query agent_templates.content.
- Call Haiku with LEVEL3_PROMPT:

```
Eres un detector de repeticion para un bot de ventas de WhatsApp.

Un check previo detecto que la plantilla candidata tiene COBERTURA PARCIAL con mensajes ya enviados.
Necesitas decidir si la plantilla agrega suficiente VALOR NUEVO para justificar enviarla.

Responde SOLO con un JSON:
{
  "decision": "ENVIAR" | "NO_ENVIAR",
  "razon": "breve explicacion de que informacion nueva agrega o por que es redundante"
}

Reglas:
- ENVIAR: La plantilla agrega al menos 1 dato o angulo que NO fue cubierto (ej: un nuevo beneficio, un dato numerico no mencionado, un argumento diferente)
- NO_ENVIAR: Todo lo que dice la plantilla ya fue comunicado en los mensajes previos, aunque con otras palabras
- Presta atencion a datos facticos (precios, tiempos, cantidades): si el precio ya se dijo, no necesita repetirse
- Si la plantilla solo "refuerza" algo ya dicho sin agregar nada, es NO_ENVIAR
```

User message format:
```
PLANTILLA CANDIDATA (texto completo):
{template.content}

MENSAJES PREVIOS RELEVANTES (texto completo):
{relevantMessages.map(m => `[${m.tipo}]: ${m.fullContent || m.tema}`).join('\n\n')}
```

- Use `claude-sonnet-4-20250514`, `max_tokens: 256`
- Parse JSON, default to 'ENVIAR' on failure (fail-open)

**Minifrase fetching helper** — private method `getTemplateMinifrase`:
- Instead of querying the DB, use the candidate template's content to derive the minifrase. The candidate templates (PrioritizedTemplate) are already loaded with content from the engine. However, PrioritizedTemplate does NOT have a minifrase field. Two options:
  a. Query agent_templates by UUID (.eq('id', templateId).select('minifrase')) — this works because templateId on PrioritizedTemplate IS the UUID PK. Add workspace_id filter: `.or('workspace_id.is.null,workspace_id.eq.{workspaceId}')`.
  b. Better: Accept workspaceId as constructor parameter: `constructor(private workspaceId?: string)` and use it in queries.
- Cache results in a Map within the filter instance (per-request lifecycle)
- Used by Level 2 for candidate templates that are in the current block (not in outbound registry yet)
- IMPORTANT: The query MUST include workspace_id filter for multi-workspace safety per CLAUDE.md Rule 3. Pass workspaceId to NoRepetitionFilter constructor from the engine.

**Orchestration in filterBlock:**
1. For each template in block:
   a. Level 1: check templatesEnviados -> if hit, filtered, continue
   b. Level 2: get candidate minifrase, call checkLevel2 -> if NO_ENVIAR, filtered, continue
   c. If PARCIAL: Level 3 checkLevel3 -> if NO_ENVIAR, filtered, continue
   d. If ENVIAR (from Level 2 or Level 3): surviving

2. Use sequential processing (not parallel) because templates share outbound context and later templates might depend on understanding earlier ones' status. Max 3 templates per block so sequential is fast enough.

3. Log each decision with level and reason.

4. Return { surviving, filtered }.

Handle edge cases:
- Empty block -> return { surviving: [], filtered: [] }
- Empty outbound registry -> all templates pass Level 1 and 2 (nothing to compare against), skip Level 2
- Template without minifrase in DB -> log warning, pass Level 2 (fail-open)
- All templates filtered -> return { surviving: [], filtered: [...] } — this is correct behavior, Plan 04 handles the empty-block case
