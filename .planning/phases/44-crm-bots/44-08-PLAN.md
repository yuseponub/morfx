<objective>
Create the two HTTP endpoints for the CRM Writer Bot: `POST /api/v1/crm-bots/writer/propose` and `POST /api/v1/crm-bots/writer/confirm`. Both enforce kill-switch + rate limit + observability wrapping. Confirm endpoint validates `actionId` uuid.

Purpose: Two-step flow requires two separate HTTP round-trips (CONTEXT D-09). A single endpoint that does both would defeat the safety property — a caller could call a "propose+confirm" in one shot. Splitting them forces deliberate double-commit semantics: propose is speculative; confirm acknowledges.

**Revision 2026-04-18 (Warnings #11, #14):** Same barrel grep precheck + x-api-key-prefix invoker fallback as Plan 07.

Output: 2 route files mirroring reader's pattern with two semantic differences:
1. Propose delegates to `writer.propose(input)` (LLM + tools via proposeAction).
2. Confirm delegates to `writer.confirm(ctx, actionId)` (no LLM — direct idempotent dispatch).

```typescript
/**
 * POST /api/v1/crm-bots/writer/propose
 *
 * CRM Writer Bot — PROPOSE phase. LLM-driven. Tool calls create proposed
 * rows in crm_bot_actions (status='proposed', 5-min TTL). Does NOT mutate
 * business entities. Caller must follow up with POST /confirm.
 * Phase 44 Plan 08.
 */

import { NextRequest, NextResponse } from 'next/server'
import { rateLimiter } from '@/lib/tools/rate-limiter'
import { runWithCollector, ObservabilityCollector, isObservabilityEnabled } from '@/lib/observability'
import { propose as writerPropose, CRM_WRITER_AGENT_ID } from '@/lib/agents/crm-writer'
import { sendRunawayAlert, maybeSendApproachingLimitAlert } from '@/lib/agents/_shared/alerts'
import { randomUUID } from 'crypto'

export async function POST(request: NextRequest): Promise<NextResponse> {
  // Kill-switch
  if (process.env.CRM_BOT_ENABLED === 'false') {
    return NextResponse.json(
      { error: 'CRM bots globally disabled', code: 'KILL_SWITCH', retryable: false },
      { status: 503 },
    )
  }

  // Header extraction — Pitfall 4 + Warning #14 fallback
  const workspaceId = request.headers.get('x-workspace-id')
  const invoker = request.headers.get('x-invoker') ?? (request.headers.get('x-api-key-prefix') ?? undefined)
  if (!workspaceId) {
    return NextResponse.json(
      { error: 'Missing workspace context', code: 'MISSING_CONTEXT', retryable: false },
      { status: 401 },
    )
  }

  // Rate limit (shared 'crm-bot' namespace with reader)
  const limit = Number(process.env.CRM_BOT_RATE_LIMIT_PER_MIN ?? 50)
  const rl = rateLimiter.check(workspaceId, 'crm-bot')
  if (!rl.allowed) {
    void sendRunawayAlert({ workspaceId, agentId: CRM_WRITER_AGENT_ID, limit })
    return NextResponse.json(
      {
        error: 'Rate limited (suspected runaway loop)',
        code: 'RATE_LIMITED',
        retryable: true,
        retry_after_ms: rl.resetMs,
      },
      { status: 429, headers: { 'Retry-After': String(Math.ceil(rl.resetMs / 1000)) } },
    )
  }
  if (rl.remaining / limit < 0.2) {
    void maybeSendApproachingLimitAlert({ workspaceId, agentId: CRM_WRITER_AGENT_ID, used: limit - rl.remaining, limit })
  }

  // Body parse
  let body: { messages?: Array<{ role: 'system' | 'user' | 'assistant'; content: string }> }
  try {
    body = await request.json()
  } catch {
    return NextResponse.json({ error: 'Invalid JSON body', code: 'INVALID_JSON', retryable: false }, { status: 400 })
  }
  const messages = body.messages ?? []
  if (!Array.isArray(messages) || messages.length === 0) {
    return NextResponse.json({ error: 'messages[] required', code: 'INVALID_INPUT', retryable: false }, { status: 400 })
  }

  // Observability-wrapped execution
  const conversationId = randomUUID()

  const exec = async () => {
    try {
      const output = await writerPropose({ workspaceId, messages, invoker })
      return NextResponse.json({ status: 'ok', output })
    } catch (err) {
      const message = err instanceof Error ? err.message : String(err)
      return NextResponse.json(
        { error: 'writer propose failed', code: 'INTERNAL', retryable: false, details: message },
        { status: 500 },
      )
    }
  }

  if (!isObservabilityEnabled()) return await exec()

  const collector = new ObservabilityCollector({
    conversationId,
    workspaceId,
    agentId: CRM_WRITER_AGENT_ID,
    turnStartedAt: new Date(),
    triggerKind: 'api',
  })
  return await runWithCollector(collector, exec)
}
```

```typescript
/**
 * POST /api/v1/crm-bots/writer/confirm
 *
 * CRM Writer Bot — CONFIRM phase. No LLM. Directly idempotent via optimistic
 * UPDATE in crm_bot_actions. The action_id must have been issued by a prior
 * propose call (within TTL=5min) for the SAME workspace.
 * Phase 44 Plan 08.
 */

import { NextRequest, NextResponse } from 'next/server'
import { rateLimiter } from '@/lib/tools/rate-limiter'
import { runWithCollector, ObservabilityCollector, isObservabilityEnabled } from '@/lib/observability'
import { confirm as writerConfirm, CRM_WRITER_AGENT_ID } from '@/lib/agents/crm-writer'
import { sendRunawayAlert, maybeSendApproachingLimitAlert } from '@/lib/agents/_shared/alerts'

const UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i

export async function POST(request: NextRequest): Promise<NextResponse> {
  if (process.env.CRM_BOT_ENABLED === 'false') {
    return NextResponse.json(
      { error: 'CRM bots globally disabled', code: 'KILL_SWITCH', retryable: false },
      { status: 503 },
    )
  }

  const workspaceId = request.headers.get('x-workspace-id')
  const invoker = request.headers.get('x-invoker') ?? (request.headers.get('x-api-key-prefix') ?? undefined)
  if (!workspaceId) {
    return NextResponse.json(
      { error: 'Missing workspace context', code: 'MISSING_CONTEXT', retryable: false },
      { status: 401 },
    )
  }

  const limit = Number(process.env.CRM_BOT_RATE_LIMIT_PER_MIN ?? 50)
  const rl = rateLimiter.check(workspaceId, 'crm-bot')
  if (!rl.allowed) {
    void sendRunawayAlert({ workspaceId, agentId: CRM_WRITER_AGENT_ID, limit })
    return NextResponse.json(
      { error: 'Rate limited', code: 'RATE_LIMITED', retryable: true, retry_after_ms: rl.resetMs },
      { status: 429, headers: { 'Retry-After': String(Math.ceil(rl.resetMs / 1000)) } },
    )
  }
  if (rl.remaining / limit < 0.2) {
    void maybeSendApproachingLimitAlert({ workspaceId, agentId: CRM_WRITER_AGENT_ID, used: limit - rl.remaining, limit })
  }

  let body: { actionId?: unknown }
  try {
    body = await request.json()
  } catch {
    return NextResponse.json({ error: 'Invalid JSON body', code: 'INVALID_JSON', retryable: false }, { status: 400 })
  }

  const actionId = body.actionId
  if (typeof actionId !== 'string' || !UUID_REGEX.test(actionId)) {
    return NextResponse.json(
      { error: 'actionId (uuid) required', code: 'INVALID_INPUT', retryable: false },
      { status: 400 },
    )
  }

  const exec = async () => {
    try {
      const result = await writerConfirm({ workspaceId, invoker }, actionId)
      return NextResponse.json({ status: 'ok', result })
    } catch (err) {
      const message = err instanceof Error ? err.message : String(err)
      return NextResponse.json(
        { error: 'writer confirm failed', code: 'INTERNAL', retryable: false, details: message },
        { status: 500 },
      )
    }
  }

  if (!isObservabilityEnabled()) return await exec()

  const collector = new ObservabilityCollector({
    conversationId: actionId,  // correlate turn to action
    workspaceId,
    agentId: CRM_WRITER_AGENT_ID,
    turnStartedAt: new Date(),
    triggerKind: 'api',
  })
  return await runWithCollector(collector, exec)
}
```

- `conversationId: actionId` gives correlation without a new schema.
- No request-body `workspaceId` is read. Action_id's workspace scope is enforced in `writerConfirm`.
- `UUID_REGEX` is defense-in-depth.

| caller → confirm | Specific action_id must match workspace from API key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-08-01 | Spoofing | caller submits action_id from another workspace | mitigate | `writerConfirm` SELECT uses `.eq('workspace_id', ctx.workspaceId)`. Cross-workspace action_id returns `not_found`. |
| T-44-08-02 | Tampering | double-confirm race | mitigate | Handled in Plan 05 two-step.ts (optimistic UPDATE). |
