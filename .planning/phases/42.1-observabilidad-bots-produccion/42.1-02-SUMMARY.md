## Type Decisions

- **Parallel to sandbox (Decision A):** Zero imports from `src/lib/sandbox/*`. Verified by `grep -r "from '@/lib/sandbox" src/lib/observability/` returning zero matches. Production system is free to evolve more deeply than the sandbox without backward-compat coupling.
- **AgentId is a closed union** of the four conversational bots in scope: `'somnio-v3' | 'godentist' | 'somnio-recompra' | 'somnio-v2'`. Robots Playwright are out of scope (Decision #1).
- **EventCategory is exhaustive** for the mechanisms enumerated in CONTEXT.md "Scope Tentativo de Captura". New categories can be added in later plans, but existing ones must remain stable for downstream UI filters.
- **`promptHash` field exists in `ObservabilityAiCall`** but the collector populates it via a local stub returning `''`. Plan 04 will replace the stub with `import { hashPrompt } from './prompt-version'` (sha256). Decoupling the type from the eventual implementation lets Wave 2 replace the stub without touching consumers.
- **`provider` field is `'anthropic'` literal** (not free string) so Plan 04 telemetry only needs to handle one provider until other LLMs join the fleet.
- **`messages` and `responseContent` typed as `unknown`** because the structures vary widely across Anthropic SDK versions and tool-use shapes. The UI panel (Plan 09+) will discriminate at render time.

## Constraints Honored

- **REGLA 6 (protect prod agent):** Default OFF (`OBSERVABILITY_ENABLED` env var must NOT be set in prod until Plan 11). Every record method wrapped in defensive try/catch so internal bugs in the observability module never crash the agent path. `flush()` is a no-op stub.
- **Pitfall 5 (no flag caching):** `flag.ts` reads `process.env.OBSERVABILITY_ENABLED` on every call — never assigned to a module-level constant. Avoids stale-flag bugs on warm Vercel functions.
- **Hot path latency budget:** Record methods are synchronous (push to in-memory arrays, no awaits, no I/O). Per CONTEXT.md, total observability latency overhead per turn must stay under ~20ms — Plan 02 contributes ~0ms because nothing actually persists yet.
- **No new npm packages:** All primitives use what is already in the repo (`node:async_hooks`, `pino`).
