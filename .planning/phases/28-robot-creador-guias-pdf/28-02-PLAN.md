2. Create `src/lib/pdf/types.ts`:
   ```typescript
   /** Input order data from domain layer (OrderForGuideGen shape) */
   export interface GuideGenOrder {
     id: string
     name: string | null
     contactName: string | null
     contactPhone: string | null
     shippingAddress: string | null
     shippingCity: string | null
     shippingDepartment: string | null
     totalValue: number
     products: Array<{ quantity: number }>
     customFields: Record<string, unknown>
     tags: string[]
   }

   /** Normalized order data (output of Claude AI) */
   export interface NormalizedOrder {
     orderId: string       // Original order ID for mapping
     numero: string        // Shipping number (order name)
     nombre: string        // First name UPPERCASE
     apellido: string      // Last name UPPERCASE
     direccion: string     // Full address
     barrio: string        // Neighborhood
     ciudad: string        // "BUCARAMANGA (STDER)"
     telefono: string      // 10-digit phone
     valorCobrar: string   // "$77.900"
     valorNumerico: number // Raw numeric value for Excel
     pagoAnticipado: boolean
     unidades: number
   }

   /** Envia Excel row data */
   export interface EnviaOrderData {
     valor: number
     nombre: string
     telefono: string
     direccion: string
     municipio: string
     departamento: string
   }
   ```

3. Create `src/lib/pdf/normalize-order-data.ts`:
   - Import Anthropic SDK (already installed: `@anthropic-ai/sdk`)
   - Export `normalizeOrdersForGuide(orders: GuideGenOrder[]): Promise<NormalizedOrder[]>`
   - Process orders in batches of 20 (per RESEARCH recommendation)
   - For each batch:
     a. Build a JSON array of order data relevant for normalization (name, contact, address, city, department, total, products, tags, custom_fields)
     b. Call Claude (model: 'claude-sonnet-4-20250514') with the normalization prompt from RESEARCH
     c. Parse the JSON response
     d. Validate each normalized order with a runtime check (all required fields present)
     e. Map back to NormalizedOrder[] with orderId preserved
   - The normalization prompt MUST include:
     - Phone: remove 57/+57/057 prefix, keep 10 digits
     - City: "CIUDAD (DEPTO_ABREV)" format, uppercase
     - Units: calculate from price ($77,900=1, $109,900=2, $139,900=3)
     - Names: UPPERCASE, split first/last
     - pagoAnticipado: true if order name contains "&" OR tags include "PAGO ANTICIPADO"
     - valorCobrar: Colombian format "$77.900"
     - barrio: extract from shipping_address if possible
   - Prompt must request JSON array response: `[{ numero, nombre, apellido, direccion, barrio, ciudad, telefono, valorCobrar, pagoAnticipado, unidades }]`
   - The function MUST handle Claude errors gracefully: if a batch fails, mark those orders as having normalization errors (return them with sensible defaults rather than throwing)
   - Export a helper `normalizedToEnvia(n: NormalizedOrder): EnviaOrderData` that converts a NormalizedOrder into the Envia Excel format (valor=number, nombre=full name, telefono, direccion, municipio=city before parenthesis, departamento=abbreviation from parenthesis)

1. Create `src/lib/pdf/generate-guide-pdf.ts`:
   - Export `generateGuidesPdf(orders: NormalizedOrder[], logoBuffer: Buffer): Promise<Buffer>`
   - Implementation:
     a. Create PDFDocument with `{ size: [288, 432], margin: 12, autoFirstPage: false }`
     b. Collect chunks via `doc.on('data', chunk => chunks.push(chunk))`
     c. For each order:
        - `doc.addPage({ size: [288, 432], margin: 12 })` — CRITICAL: pass size to each addPage
        - Logo: `doc.image(logoBuffer, 12, 12, { width: 80 })`
        - Shipping number (order.numero): fontSize 14, bold, right-aligned at top
        - Recipient block starting at y=60:
          - Name: `${order.nombre} ${order.apellido}` bold
          - Address: order.direccion
          - City: `${order.barrio} - ${order.ciudad}`
          - Phone: `Tel: ${order.telefono}`
        - Amount: order.valorCobrar, fontSize 16, bold, centered
        - Units indicator: `UNIDADES: ${order.unidades}` if unidades > 1
        - Barcode (Code 128): use bwip-js toBuffer({ bcid: 'code128', text: order.numero, scale: 2, height: 12, includetext: true, textxalign: 'center' }) then doc.image(barcodePng, ...)
        - Pago anticipado: if order.pagoAnticipado, draw "PAGO ANTICIPADO" at bottom
     d. `doc.end()` then resolve Promise with `Buffer.concat(chunks)`
   - PITFALL: Wrap in Promise and resolve on 'end' event (not 'finish')
   - PITFALL: bwip-js toBuffer is async, so each page generation is async
   - Use try/catch around bwip-js per order — if barcode fails, skip barcode but still generate the label

2. Create `src/lib/pdf/generate-envia-excel.ts`:
   - Export `generateEnviaExcel(orders: EnviaOrderData[]): Promise<Buffer>`
   - Implementation:
     a. Create ExcelJS Workbook and Worksheet named 'Envios Envia'
     b. Define columns: Valor (width 12), Nombre (30), Telefono (15), Direccion (40), Municipio (20), Departamento (18)
     c. Style header row: bold font, light gray background
     d. Add one row per order
     e. Use `workbook.xlsx.writeBuffer()` and convert to Buffer
   - This is the simpler of the two generators
