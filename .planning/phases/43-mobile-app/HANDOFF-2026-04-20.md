## Bugs pendientes (aplicar en la siguiente wave antes/durante Plan 15)

### 1. Template send falla con error "internal" (P0 para Plan 15 wave)
**Reportado:** 2026-04-20 tras verificar Plan 14 attach menu.
**Síntoma:** usuario escoge plantilla WhatsApp desde el menú de adjuntar → llena variables → Send → aparece toast/alerta "No se pudo enviar plantilla: internal" (el 'internal' sugiere 500 del server).
**Dónde investigar:**
- Server: `src/app/api/mobile/conversations/[id]/messages/route.ts` — buscar la rama que maneja `templateName` + `templateVariables`. El error 'internal' viene de `toMobileErrorResponse` default path (status 500).
- Probable causa: el domain `src/lib/domain/messages-send-idempotent.ts` no soporta templates, O el dispatcher a Meta Graph API falla porque la plantilla no está registrada en Meta Business Manager (recuerda que Phase 37.5 / 38 aún no están completas — Meta integration no está fully live).
- Fallback: si Meta direct integration no está lista, posiblemente el send de plantillas requiere todavía el provider 360dialog O un simulador. Chequear `src/lib/whatsapp/` para ver qué provider está activo.
**Alternativa:** documentar como "pendiente hasta Plan 38/39 (Meta outbound templates)" si no se puede resolver sin esos plans.

**Síntoma:** typing en SearchBar dispara fetch (confirmado), pero no se ven resultados en la UI.
**Dónde investigar:**

**Pendiente:** aplicar con el siguiente wave.
