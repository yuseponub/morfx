  <action>
  Create migration file `supabase/migrations/20260226000000_block_priorities.sql` with:

  1. **Add priority column to agent_templates:**
  ```sql
  ALTER TABLE agent_templates
    ADD COLUMN priority TEXT NOT NULL DEFAULT 'CORE'
    CHECK (priority IN ('CORE', 'COMPLEMENTARIA', 'OPCIONAL'));
  ```

  2. **Seed priorities based on existing orden values:**
  ```sql
  -- orden=0 is always CORE (direct answer to the intent)
  -- orden=1 is COMPLEMENTARIA (supporting info)
  -- orden=2+ is OPCIONAL (nice-to-have)
  UPDATE agent_templates SET priority = 'CORE' WHERE orden = 0;
  UPDATE agent_templates SET priority = 'COMPLEMENTARIA' WHERE orden = 1;
  UPDATE agent_templates SET priority = 'OPCIONAL' WHERE orden >= 2;
  ```
  The DEFAULT 'CORE' already handles orden=0, but explicit UPDATE for clarity.

  3. **Add pending_templates JSONB column to session_state:**
  ```sql
  ALTER TABLE session_state
    ADD COLUMN pending_templates JSONB NOT NULL DEFAULT '[]'::jsonb;
  ```

  4. **Add comment on new columns:**
  ```sql
  COMMENT ON COLUMN agent_templates.priority IS 'Template priority: CORE (must send), COMPLEMENTARIA (supporting), OPCIONAL (nice-to-have, dropped first)';
  COMMENT ON COLUMN session_state.pending_templates IS 'Templates not sent due to interruption, stored for merge into next block. Cleared on HANDOFF or session close.';
  ```

  5. **Update the UNIQUE constraint** to keep working (priority is not part of uniqueness — same intent+visit_type+orden+workspace can only have one priority).

  No index needed on pending_templates (only accessed by session_id PK lookup).
