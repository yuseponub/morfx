## Option B — DevTools console snippet (reliable, use this one for Task 2)

This is the recommended path. The user is already logged in in the browser, so the
authenticated /whatsapp route renders naturally.

### Step-by-step

1. Start the dev server: `npm run dev` (port 3020).
2. Log in to http://localhost:3020 with a user that belongs to a test workspace
   (NOT a productive customer workspace — Regla 6).
3. Flip the flag for the test workspace (SQL snippet — see 06-PLAN.md Task 2 Step 3).
4. Navigate to http://localhost:3020/whatsapp. Wait for the inbox to fully render
   (conversation list + chat view + contact panel).
5. Open DevTools (F12) → Console tab.
6. Paste the snippet below and press Enter.

### Snippet

```javascript
// axe-core 4.8.2 via cloudflare CDN (no install needed).
// Runs wcag2a + wcag2aa rule sets across the /whatsapp page.
// Filters to serious/critical impact violations (per UI-SPEC §16 DoD item 9).
(() => {
  const s = document.createElement('script');
  s.src = 'https://cdnjs.cloudflare.com/ajax/libs/axe-core/4.8.2/axe.min.js';
  s.onload = () => {
    axe.run({ runOnly: ['wcag2a', 'wcag2aa'] }).then(r => {
      const all = r.violations;
      const serious = all.filter(v => v.impact === 'serious' || v.impact === 'critical');
      console.log('=== axe-core scan — /whatsapp (flag ON) ===');
      console.log('Total violations:', all.length);
      console.log('Serious/critical (BLOCKING per DoD §16.9):', serious.length);
      if (serious.length > 0) {
        console.table(serious.map(v => ({
          id: v.id,
          impact: v.impact,
          description: v.description,
          nodes: v.nodes.length,
          helpUrl: v.helpUrl
        })));
      } else {
        console.log('✅ PASS — zero serious/critical violations.');
      }
      // Also log minor/moderate for the record (non-blocking)
      const minorModerate = all.filter(v => v.impact === 'minor' || v.impact === 'moderate');
      console.log('Minor/moderate (non-blocking, informational):', minorModerate.length);
    });
  };
  document.head.appendChild(s);
})();
```

### Expected output (PASS case)

```
=== axe-core scan — /whatsapp (flag ON) ===
Total violations: N              // any integer ≥ 0
Serious/critical (BLOCKING per DoD §16.9): 0
✅ PASS — zero serious/critical violations.
Minor/moderate (non-blocking, informational): M
```

### Failure case — what to do

If `Serious/critical (BLOCKING per DoD §16.9)` is greater than 0:

1. Inspect the console.table output — each row has `id` (rule name), `description`, and `helpUrl`.
2. Fix the violation in the relevant TSX file under `src/app/(dashboard)/whatsapp/components/`
   (most likely missing aria-label, wrong heading order, insufficient contrast, or missing
   alt text — Plans 02/04/05 added aria-labels universally, so this should be rare).
3. Re-run the snippet.
4. Only proceed to Task 3 once the count is 0.
