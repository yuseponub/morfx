Add a `ProductMappingEditor` sub-component in actions-step.tsx (place it after the existing TemplateVarRow component, before DelayEditor). Then add a custom product mode section in ActionCard specifically for create_order actions.

**ProductMappingEditor component:**

```typescript
interface ProductMappingConfig {
  source: string
  mappings: Array<{ when: string; productId: string; quantity: number }>
  defaultProductId?: string | null
  defaultQuantity?: number
}

function ProductMappingEditor({
  config,
  onChange,
  products,
  triggerType,
}: {
  config: ProductMappingConfig
  onChange: (config: ProductMappingConfig) => void
  products: Product[]
  triggerType: TriggerType
})
```

UI structure (mirrors TemplateVarRow conditional mode pattern):
1. **Source variable section**: Label "Variable fuente" + Input with VariablePicker button (same pattern as TemplateVarRow line 278-295). Placeholder: "Ej: {{shopify.total}}"
2. **Mapping rows section**: Label "Si el valor es... entonces asignar:" + for each mapping:
   - Input "Si es..." for `when` value (text, flex-1)
   - Arrow separator "→"
   - Select dropdown for product (flex-1) — shows `product.title ($product.price)` format, value is product.id. Add placeholder "Seleccionar producto..."
   - Trash2 button to remove mapping
   - Each row has a small "Cant:" label + number Input (w-16, default 1, min 1) for quantity
3. **Add button**: "Agregar condicion" button (same style as TemplateVarRow line 337-346)
4. **Default product section**: Label "Producto por defecto" + Select dropdown for product (same product list, optional — add empty option "Ninguno"). Below: small "Cant:" label + number Input for defaultQuantity.

Use the same Tailwind classes as TemplateVarRow for consistency:
- Container: `rounded-md border bg-muted/30 p-2 space-y-2`
- Labels: `text-[10px]`
- Inputs: `h-7 text-xs`
- Buttons: same ghost/outline patterns

**Product mode section in ActionCard:**

Inside the ActionCard component, add a custom section that renders BEFORE the standard params loop. Check `if (action.type === 'create_order')` and render a product mode selector.

Implementation:
1. Compute current mode:
```typescript
type ProductMode = 'none' | 'copy' | 'conditional'
function getProductMode(params: Record<string, unknown>): ProductMode {
  if (params.productMappings) return 'conditional'
  if (params.copyProducts) return 'copy'
  return 'none'
}
```
2. Render a section with Label "Productos" and 3 radio-style buttons (using existing shadcn pattern — could be 3 small buttons with active state, or a simple Select with 3 options):
   - "Sin productos" (none)
   - "Copiar del trigger" (copy)
   - "Asignar por condicion" (conditional)

   Use a row of 3 small buttons with `cn()` for active/inactive state. Active: `bg-primary text-primary-foreground`. Inactive: `bg-muted text-muted-foreground hover:bg-muted/80`.

3. Mode change handler:
   - Switch to 'none': set `copyProducts: false`, remove `productMappings`
   - Switch to 'copy': set `copyProducts: true`, remove `productMappings`
   - Switch to 'conditional': set `copyProducts: false`, set `productMappings: { source: '', mappings: [{ when: '', productId: '', quantity: 1 }], defaultProductId: null, defaultQuantity: 1 }`
   - Use `updateParam` for each change. For removing a param, set it to `undefined`.

4. When mode is 'conditional', render `<ProductMappingEditor>` below the buttons.

**CRITICAL placement in ActionCard:** The product mode section should appear inside the `<div className="space-y-3 border-t pt-3">` params section, but BEFORE the standard params loop. This way it replaces the `copyProducts` boolean toggle with the 3-mode selector.

**Hide copyProducts from standard params:** When action.type is 'create_order', filter out the `copyProducts` param from both the required params loop AND the optional params so it doesn't render as a duplicate toggle. Do this by adding a filter:
```typescript
const hiddenParams = action.type === 'create_order' ? ['copyProducts', 'productMappings'] : []
```
Then add `.filter(p => !hiddenParams.includes(p.name))` to both the required and optional params filter chains.

**Import Package from lucide-react** for the product mode section icon if desired (optional — can also use ShoppingCart already imported).
