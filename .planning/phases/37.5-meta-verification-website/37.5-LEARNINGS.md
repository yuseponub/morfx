## Bugs Found & Resolved During Execution

### 1. Middleware composition breaking auth bypass (Plan 01 deviation — Rule 3)

**Symptom:** Initial approach of creating `src/middleware.ts` would have duplicated repo-root middleware and lost critical bypass paths (`/api/inngest`, `/api/cron`, `/_next/*`).

**Fix:** Edited repo-root `middleware.ts` in place. Composed next-intl middleware with existing `updateSession` Supabase whitelist: 6 marketing paths go through `createMiddleware()` first, everything else falls through to `updateSession()`.

**Commit:** `8cd69c7 feat(37.5-01): compose next-intl middleware for marketing routes + whitelist`

### 2. Parallel plan execution clobbering shared JSON files (Plans 03 + 04)

**Symptom:** Plan 03 (Landing namespace) and Plan 04 (Terms + Privacy namespaces) both wrote to `messages/es.json` and `messages/en.json` in parallel waves. Heredoc-based merges collided.

**Fix:** Atomic re-read-merge pattern via Python — `json.load` → modify dict → `json.dump` in a single process invocation. No heredocs, no template literals, no interleaved stdout. This pattern should become standard for parallel plans editing shared structured files.

### 3. LegalSection missing recursive subsection support (Plan 04 deviation — Rule 2)

**Symptom:** Privacy section 7.2 (Finalidades del Tratamiento) requires nested subsections under each finalidad. Original `LegalSection.tsx` supported only one level of nesting.

**Fix:** Added recursive rendering: a section's `subsections` can itself contain `subsections`. Rendered with progressive indentation. Added inline during Plan 04 Task 2.

### 4. Pre-existing React 19 peer dep incompatibility

**Symptom:** `npm install next-intl` failed without `--legacy-peer-deps` because `@webscopeio/react-textarea-autocomplete` has a React 18 peer dep while the app uses React 19.2.3.

**Fix (not this phase):** Used `--legacy-peer-deps`. Pre-existing issue unrelated to Phase 37.5. Documented here so future plans don't re-diagnose.

### 5. WSL + Geist fonts network outage blocking local build

**Symptom:** `npm run build` times out fetching Geist font files from `https://fonts.cdnfonts.com/s/*` during Next.js font optimization. Pattern documented in Plans 42.1-07/09/10 summaries.

**Mitigation:** Skipped local `npm run build` during regression (Plan 05 Task 1). Relied on Vercel build pipeline instead — Vercel has direct network access to font CDN. `npx tsc --noEmit` still validates types.

## Patterns Learned

### next-intl 4.x with Next.js 16

- `localePrefix: 'as-needed'` → default locale (ES) served at `/`, other locales (EN) at `/en/*`. No redirect for root.
- Route group `(marketing)/[locale]/` scopes locale to public pages only. Auth (`/login`, `/signup`), dashboard (`/crm`), APIs stay outside the locale segment entirely.
- `params` is now a Promise: `async function Page({ params }: { params: Promise<{ locale: string }> }) { const { locale } = await params; }`. Forgetting the `await` produces subtle hydration mismatches.
- `messages/` folder at repo root (not `src/messages/`) per next-intl convention; the `getMessages()` request config reads from there.
- `NextIntlClientProvider` wraps only marketing layout children. Dashboard/auth layouts do NOT import it.

### Middleware composition (critical for mixed public/auth apps)

- **Order matters:** marketing path check → `createMiddleware(nextIntl)` → else → `updateSession(supabase)`.
- Marketing paths: `/`, `/en`, `/privacy`, `/terms`, `/en/privacy`, `/en/terms` (hardcoded whitelist, not regex — explicit is better than matched here).
- `matcher` in middleware config must exclude `_next/*`, `*.png`, `api/*` as before — the intl middleware doesn't need to see static assets either.

### Route group `(marketing)/` keeps locale scoped

- `src/app/(marketing)/[locale]/layout.tsx` sets up `NextIntlClientProvider` + Header + Footer
- `src/app/(marketing)/[locale]/page.tsx` renders landing (composed from `src/components/marketing/*`)
- `src/app/(marketing)/[locale]/privacy/page.tsx` + `terms/page.tsx` render legal docs via `LegalSection` component
- Auth pages (`src/app/login/page.tsx`, etc.) are siblings to `(marketing)` — no locale param, no intl provider

### Handwritten privacy + terms beats template services

- Source: user's legal team provided a complete T&C document (`reference/TyC-original.docx`)
- Transcribed to JSON structure keyed by `Terms.section.N.*` and `Privacy.section.N.*`
- Named stack explicitly (Supabase, Vercel, Meta, Alegra, Inngest) — keeps legal content aligned with actual infrastructure
- No dependency on Termly/iubenda — no template drift, no paid service
- 4 corrections applied during transcription: NIT `902.058.328-5` → `902.052.328-5`, casing to `MORFX S.A.S.`, removed 360dialog (Meta Direct only now), kept Gmail until Block B

### Meta rejection root cause analysis (high-confidence, multi-source)

Online research during Phase 37.5 discussion converged on 4 causes for Business Verification rejection in 2026:

1. **Login-only domain** — reviewer lands on `/login`, sees no business content, cannot verify legitimacy
2. **No legal name visible on website** — reviewer can't cross-check Cámara de Comercio vs. website owner
3. **Personal-domain email** (Gmail/Hotmail/Yahoo) as business contact — screams "not a real business"
4. **Domain not verified in Business Manager** — reviewer's tooling flags mismatch between portfolio domain and actual website owner

Fixing any one alone is insufficient. Shotgun approach: fix all 4 before resubmitting. Phase 37.5 fixes 1, 2, 4. Block B fixes 3.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| ES as default, EN at `/en` | MORFX is Colombian; Meta reviewer sees ES first which matches Cámara de Comercio doc language |
| Single OG image for both locales | 1 branded asset, less maintenance, same visual brand regardless of locale |
| Hand-written legal docs (not Termly/iubenda) | Alignment with real stack; no monthly subscription; team legal-reviewed source |
| Footer corporate email stays Gmail until Block B | Avoids referencing `contacto@morfx.app` / `info@morfx.app` before it exists; would be a dead link if rushed |
| Handle Block B/C in separate instance | Block A (website) is self-contained; parallelizing Block B (email forwarding) and Block C (Facebook Page) avoids blocking Meta resubmission on unrelated work |
| Skip local `npm run build` during Task 1 regression | WSL Geist fonts outage is pre-existing and blocks build deterministically; Vercel pipeline validates build correctly |
| Middleware lives at repo root, not `src/` | Preserves existing bypass paths for Inngest/cron; new paths added in-place rather than duplicated |
| `localePrefix: 'as-needed'` | Root `/` renders ES landing (cleaner for Meta reviewer + Colombian users) while still supporting `/en` for English |
