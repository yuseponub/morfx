# Phase 37.5 Plan 01: Middleware + i18n Foundation Summary

**One-liner:** next-intl v4 wired with ES/EN routing + composed middleware exempts 6 public marketing routes from Supabase auth redirect, unblocking Meta domain verification.

## What Was Built

Foundation layer so the (marketing) route group in Plan 02 can serve the landing + privacy + terms pages publicly. Without this plan, the Supabase middleware hard-redirected everything outside its 7-route whitelist to `/login`, which is exactly why Meta Business Verification rejected morfx.app silently on 2026-03-31 (reviewers only saw a login screen).

### Task 1 — Install next-intl 4.x + plugin wiring

- `npm install next-intl@^4 --legacy-peer-deps` -> `next-intl@^4.9.1` in package.json.
- `next.config.ts` wrapped with `withNextIntl = createNextIntlPlugin('./src/i18n/request.ts')`, preserving turbopack.root, serverExternalPackages (pdfkit, bwip-js), experimental.serverActions.bodySizeLimit, and images.remotePatterns.
- Commit: `44414de`

### Task 2 — Routing + request config

- `src/i18n/routing.ts`: `defineRouting({ locales: ['es','en'], defaultLocale: 'es', localePrefix: 'as-needed' })`.
- `src/i18n/request.ts`: `getRequestConfig` with `hasLocale` guard, dynamic import of `../../messages/${locale}.json`.
- `messages/es.json` and `messages/en.json` created at repo root with `{}` (real content lands in Plans 02-04).
- Commit: `5bf2b0b`

### Task 3 — Composed middleware + whitelist (before/after)

**Before** (`middleware.ts` root):
```ts
import { type NextRequest, NextResponse } from 'next/server'
import { updateSession } from '@/lib/supabase/middleware'
// ... bypasses para /api/webhooks, /api/inngest, /api/manychat, /api/mobile, /api/v1/tools ...
return await updateSession(request)
```

**After** (`middleware.ts` root, first check added before all other bypasses):
```ts
import createMiddleware from 'next-intl/middleware'
import { routing } from '@/i18n/routing'

const intlMiddleware = createMiddleware(routing)
const PUBLIC_MARKETING_ROUTES = new Set([
  '/', '/en', '/privacy', '/en/privacy', '/terms', '/en/terms',
])

// First check:
if (PUBLIC_MARKETING_ROUTES.has(pathname)) {
  return intlMiddleware(request)
}
// ... resto de bypasses y updateSession intactos ...
```

**Before** (`src/lib/supabase/middleware.ts` publicRoutes, lines 38-46):
```ts
const publicRoutes = [
  '/login', '/signup', '/forgot-password', '/reset-password',
  '/auth/callback', '/auth/confirm', '/invite',
]
```

**After** (publicRoutes extended with 6 marketing routes):
```ts
const publicRoutes = [
  '/login', '/signup', '/forgot-password', '/reset-password',
  '/auth/callback', '/auth/confirm', '/invite',
  // Marketing routes (Phase 37.5) — bypassed by composed middleware,
  // listed here as defense-in-depth.
  '/', '/privacy', '/terms', '/en', '/en/privacy', '/en/terms',

```

## Deviations from Plan

### Rule 3 (blocking) deviations — no permission needed, documented for traceability

**1. [Rule 3 - Blocking] Modified existing `middleware.ts` at repo root instead of creating `src/middleware.ts`**

- **Found during:** Task 3
- **Issue:** PLAN instructs creating `src/middleware.ts`. Repo already has `middleware.ts` at root with critical bypasses: `/api/webhooks`, `/api/inngest`, `/api/manychat`, `/api/mobile`, `/api/temp-send-agendados`, `/api/v1/tools` (API-key validation). Next.js only loads middleware from ONE location. Creating `src/middleware.ts` would either have been shadowed (no effect) or dropped all those bypasses (production incident).
- **Fix:** Updated the existing file in-place. Prepended next-intl check before all other logic. Kept every pre-existing branch byte-identical below.
- **Files modified:** `middleware.ts` (root)
- **Commit:** `8cd69c7`

**2. [Rule 3 - Blocking] `npm install --legacy-peer-deps` required**

- **Found during:** Task 1
- **Issue:** Pre-existing peer-dep conflict between `@webscopeio/react-textarea-autocomplete@4.9.2` (peer `react@16||17||18`) and `react@19.2.3`. ANY `npm install` in this repo fails without `--legacy-peer-deps`. Not introduced by this plan.
- **Fix:** Used `--legacy-peer-deps` flag.
- **Follow-up (out of scope):** Someone should audit whether that autocomplete package can be upgraded / replaced so future installs don't need the flag. Not blocking Phase 37.5.

### No Rule 1/2/4 deviations — plan executed as written otherwise.
