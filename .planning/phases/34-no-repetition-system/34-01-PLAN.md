**File 1: `no-repetition-types.ts`** — Shared types for the entire no-repetition system.

Define and export:

```typescript
import type { PrioritizedTemplate } from './block-composer'

/** A single outbound message entry in the conversation registry */
export interface OutboundEntry {
  tipo: 'plantilla' | 'humano' | 'ia'
  id: string | null          // template ID for plantilla, null for human/AI
  tema: string               // minifrase (from DB field or generated by Haiku)
  fullContent?: string       // Only loaded for Level 3 checks (lazy)
}

/** Level 2 Haiku decision for minifrase comparison */
export type NoRepLevel2Decision = 'ENVIAR' | 'NO_ENVIAR' | 'PARCIAL'

/** Result of filtering a single template through the 3-level system */
export interface FilteredTemplateEntry {
  template: PrioritizedTemplate
  level: 1 | 2 | 3
  reason: string
}

/** Result of the full no-repetition filter on a block */
export interface NoRepFilterResult {
  /** Templates that passed all checks — send these */
  surviving: PrioritizedTemplate[]
  /** Templates blocked by no-repetition — do NOT send */
  filtered: FilteredTemplateEntry[]
}
```

**File 2: `outbound-registry.ts`** — Builds the outbound registry from existing DB tables.

Create a function `buildOutboundRegistry` that reconstructs all outbound messages for a conversation:

```typescript
import { createAdminClient } from '@/lib/supabase/admin'
import { createModuleLogger } from '@/lib/audit/logger'
import type { OutboundEntry } from './no-repetition-types'

const logger = createModuleLogger('outbound-registry')

/**
 * Build the outbound registry for a conversation.
 * Reconstructs all sent messages (templates, human, AI) from existing DB tables.
 *
 * Sources:
 * 1. templates_enviados (string[]) + agent_templates.minifrase -> plantilla entries
 * 2. messages WHERE direction='outbound' AND conversation_id -> outbound entries
 * 3. agent_turns WHERE role='assistant' AND session_id -> AI entries
 *
 * Human vs AI disambiguation: outbound messages that DON'T match any agent_turn content
 * are classified as 'humano'. Messages that match agent_turn content are 'ia'.
 *
 * @param conversationId - Conversation to build registry for
 * @param sessionId - Agent session ID (for agent_turns query)
 * @param templatesEnviados - Template IDs already sent (from session_state)
 * @returns Array of OutboundEntry with tipo, id, and tema
 */
export async function buildOutboundRegistry(
  conversationId: string,
  sessionId: string,
  templatesEnviados: string[]
): Promise<OutboundEntry[]>
```

Implementation details:
1. Query `agent_templates` for the minifrases of templatesEnviados IDs:
   ```typescript
   const { data: templates } = await supabase
     .from('agent_templates')
     .select('id, minifrase, content')
     .in('id', templatesEnviados)
   ```
   Create OutboundEntry for each with tipo='plantilla', tema from minifrase field.

2. Query outbound messages from `messages` table:
   ```typescript
   const { data: outboundMsgs } = await supabase
     .from('messages')
     .select('id, content, timestamp')
     .eq('conversation_id', conversationId)
     .eq('direction', 'outbound')
     .order('timestamp', { ascending: true })
   ```

3. Query assistant turns from `agent_turns`:
   ```typescript
   const { data: aiTurns } = await supabase
     .from('agent_turns')
     .select('id, content, created_at')
     .eq('session_id', sessionId)
     .eq('role', 'assistant')
     .order('created_at', { ascending: true })
   ```

4. Identify human messages: outbound messages whose content does NOT appear in any agent_turn content. Create OutboundEntry with tipo='humano', id=null, tema='', **fullContent=message.content**. The tema will be filled by minifrase generator in Plan 02, but fullContent MUST be populated here so Level 3 can access the full text.

5. Identify AI messages: agent_turn content that is NOT a known template content. Create OutboundEntry with tipo='ia', id=null, tema='', **fullContent=aiTurn.content**. Same rationale: tema is generated later, but fullContent is needed for Level 3.

6. Return combined array, log count per tipo.

IMPORTANT: The tema field for human/AI messages is left empty ('') here. Plan 02's minifrase generator will populate them. However, `fullContent` MUST be populated from the source content (message.content for human, aiTurn.content for AI) so that:
  - Plan 02's minifrase generator can read fullContent to generate minifrases
  - Plan 02's Level 3 filter can access full text for deep comparison

Handle edge cases:
- Empty templatesEnviados -> skip template query
- No outbound messages -> return empty array for human/AI entries
- Guard against null content in messages/turns
