Create a module that paraphrases template content via Claude for repeated intents.

Follow the Anthropic SDK direct call pattern from message-classifier.ts.

**PARAPHRASE_PROMPT (system prompt):**
```
Eres un asistente de ventas de WhatsApp para Somnio (suplemento natural para dormir con melatonina y magnesio).

El cliente ya recibio esta informacion antes. Necesitas PARAFRASEAR el siguiente mensaje para que suene fresco y natural, como si fuera la primera vez.

REGLAS CRITICAS:
1. MANTENER todos los datos facticos exactos (precios, numeros, tiempos, cantidades, ingredientes)
2. Solo cambiar estructura, orden de oracion, y expresiones
3. Tono amigable y colombiano (tutear, informal)
4. Maximo 20% mas corto que el original (nunca mas largo)
5. NO agregar informacion nueva que no este en el original
6. NO usar emojis a menos que el original los tenga
7. NO agregar saludos ni despedidas

Responde SOLO con el mensaje parafraseado, sin explicaciones ni comillas.
```

**Export function:**
```typescript
import Anthropic from '@anthropic-ai/sdk'
import { createModuleLogger } from '@/lib/audit/logger'

const logger = createModuleLogger('template-paraphraser')

/**
 * Paraphrase a template's content so it sounds fresh for repeated intents.
 * Preserves all factual data (prices, numbers, times, quantities).
 *
 * @param originalContent - The original template content text
 * @returns Paraphrased content string
 */
export async function paraphraseTemplate(originalContent: string): Promise<string>
```

Implementation:
1. Create Anthropic client (or reuse module-level singleton).
2. Call `claude-sonnet-4-20250514` with `max_tokens: 512` (paraphrased text can be up to original length).
3. User message: the original content text directly.
4. Extract text from response (same extractText pattern as message-classifier.ts).
5. Validate result:
   - Not empty
   - Not longer than original * 1.3 (safety check — prompt says 20% shorter max, but allow some tolerance)
   - If validation fails, return original content unchanged (fail-safe: original is better than bad paraphrase)
6. Log original length vs paraphrased length.

Handle edge cases:
- Very short templates (< 20 chars) -> return original (not worth paraphrasing)
- API error -> return original content (fail-safe)
- Empty response -> return original content

**Changes to `getTemplatesForIntent`:**

The existing flow is:
```
1. Determine visitType (primera_vez or siguientes)
2. Filter templates by intent + visitType
3. Fallback: if no siguientes, use primera_vez
4. Filter out already-sent templates
5. Sort by orden
```

New flow:
```
1. Determine if this is a repeated visit (intent already in intentsVistos)
2. ALWAYS query visit_type='primera_vez' (siguientes no longer exist)
3. Filter out already-sent templates
4. Sort by orden
5. If repeated visit:
   a. Cap at top 2 by priority (CORE first, then COMP, then OPC)
   b. Paraphrase each surviving template via paraphraseTemplate()
6. Return TemplateSelection with isRepeatedVisit flag
```

Specific code changes:

a. **Remove siguientes logic:** Delete the `visitType` determination and the fallback block (lines ~109-121 in current code). Always filter by `visit_type === 'primera_vez'`.

b. **Add repeated visit detection:** Use the existing `isFirstVisit()` method. If NOT first visit, set `isRepeated = true`.

c. **Cap repeated intents at 2 templates:** When `isRepeated`, after filtering out already-sent templates, take only the top 2 by priority. Use the PRIORITY_RANK from block-composer.ts for sorting:
   ```typescript
   if (isRepeated) {
     // Sort by priority (CORE first), then by orden
     const PRIORITY_RANK = { CORE: 0, COMPLEMENTARIA: 1, OPCIONAL: 2 }
     templatesNotSent.sort((a, b) => {
       const rankDiff = (PRIORITY_RANK[a.priority] ?? 2) - (PRIORITY_RANK[b.priority] ?? 2)
       if (rankDiff !== 0) return rankDiff
       return a.orden - b.orden
     })
     templatesNotSent = templatesNotSent.slice(0, 2) // Max 2 for repeated intents
   }
   ```

d. **Paraphrase repeated templates in processTemplates:** Update `processTemplates` to accept an `isRepeated` flag. When true, call `paraphraseTemplate()` on each template's content AFTER variable substitution:
   ```typescript
   processTemplates(
     templates: AgentTemplate[],
     context: VariableContext,
     isRepeated: boolean = false
   ): Promise<ProcessedTemplate[]>
   ```
   NOTE: This changes the return type from sync to async (`Promise<ProcessedTemplate[]>`). Update all callers accordingly. The method now needs `async` because paraphrasing is async.

e. **Update TemplateSelection type:** Add `isRepeatedVisit: boolean` field so callers know whether paraphrasing was applied.

f. **Update the `rowToTemplate` method:** No changes needed (it already maps all fields correctly).

g. **Update `getTemplatesForIntents` (plural):** Pass through the isRepeated detection.

IMPORTANT: The `processTemplates` return type change from `ProcessedTemplate[]` to `Promise<ProcessedTemplate[]>` will require updating callers. The only caller is in `somnio-orchestrator.ts` (around line 540-550). Plan 04 handles the orchestrator update. For now, just update the TemplateManager and note the breaking change.

Handle edge cases:
- 0 templates survive filter for repeated intent -> return empty (correct behavior, Plan 04 handles)
- Paraphrase fails -> processTemplates returns original content (paraphraseTemplate is fail-safe)
- All templates already sent for repeated intent -> return empty
