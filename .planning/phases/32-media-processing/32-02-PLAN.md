**audio-transcriber.ts:**

Create `transcribeAudioFromUrl(audioUrl: string, mimeType: string)` that returns `Promise<{ success: true; text: string } | { success: false; error: string }>`.

Implementation:
1. Import `OpenAI` and `toFile` from `'openai'`
2. Create client: `new OpenAI({ apiKey: process.env.OPENAI_API_KEY })`
3. Fetch audio buffer from Supabase Storage URL (`fetch(audioUrl)`)
4. Determine file extension from mimeType:
   - `audio/ogg` or contains 'ogg' -> `.ogg`
   - `audio/mpeg` or contains 'mpeg' -> `.mp3`
   - `audio/aac` or contains 'aac' -> `.aac`
   - `audio/amr` -> `.amr`
   - Default: `.ogg` (WhatsApp voice notes default)
5. Convert to file: `await toFile(buffer, 'voice' + ext, { type: mimeType || 'audio/ogg' })`
6. Call Whisper: `openai.audio.transcriptions.create({ file, model: 'whisper-1', language: 'es' })`
7. If transcription.text is empty or whitespace-only, return `{ success: false, error: 'Empty transcription' }`
8. Return `{ success: true, text: transcription.text.trim() }`
9. Wrap everything in try/catch -- on ANY error, return `{ success: false, error: errorMessage }`

IMPORTANT: Do NOT write to filesystem. Vercel serverless has no persistent /tmp. Use `toFile()` from the openai SDK which creates an in-memory File object from a buffer.

**sticker-interpreter.ts:**

Create `interpretSticker(stickerUrl: string)` that returns `Promise<{ gesto: string | null; descripcion: string }>`.

Implementation:
1. Import `Anthropic` from `'@anthropic-ai/sdk'` (already installed, used by OCR module)
2. Create a `STICKER_VISION_PROMPT` constant (Spanish). The prompt should:
   - Explain the role: "interpreter de stickers de WhatsApp"
   - List recognized gestures and their text equivalents: ok, hola, jaja, gracias
   - Instruct to respond with JSON: `{"gesto": "ok" | "hola" | "jaja" | "gracias" | null, "descripcion": "..."}`
   - Emphasize: if the sticker does NOT clearly express one of the basic gestures, gesto MUST be null
3. Create a `RECOGNIZED_GESTURES` Set: `new Set(['ok', 'hola', 'jaja', 'gracias'])`
4. Fetch sticker from URL, convert to base64 (`Buffer.from(await res.arrayBuffer()).toString('base64')`)
5. Call Claude Vision: `client.messages.create({ model: 'claude-sonnet-4-6', max_tokens: 256, messages: [{ role: 'user', content: [image block (base64, media_type: 'image/webp'), text block with prompt] }] })`
6. Parse JSON from response text (same pattern as `src/lib/ocr/extract-guide-data.ts`): extract text blocks, find JSON with regex `/\{[\s\S]*\}/`, parse, validate gesto is in RECOGNIZED_GESTURES
7. On ANY failure (fetch, API, parse), return `{ gesto: null, descripcion: 'Error: ...' }`

Use `claude-sonnet-4-6` NOT `claude-haiku-3.5-sonnet` -- matches the existing OCR module pattern. The cost per sticker is ~$0.001-0.005 (acceptable).

Handle edge case: sticker mimeType could be 'image/webp' (standard) or 'image/png' (rare). If the URL doesn't indicate webp, use the actual Content-Type from the fetch response. Pass `media_type` accurately to Claude Vision.

**media-gate.ts:**

Create `processMediaGate(input: MediaGateInput): Promise<MediaGateResult>` as the main entry point.

Implementation (switch on input.messageType):

1. `'text'` -> Return `{ action: 'passthrough', text: input.messageContent }` immediately.

2. `'audio'` -> Call `handleAudio(input)`:
   - If no mediaUrl, return handoff (can't transcribe without audio file)
   - Call `transcribeAudioFromUrl(input.mediaUrl, input.mediaMimeType || 'audio/ogg')`
   - If success, return `{ action: 'passthrough', text: result.text }`
   - If failure, return `{ action: 'handoff', reason: 'No se pudo transcribir el audio del cliente' }`
   - Log transcription result (success/failure, character count) using createModuleLogger('media-gate')

3. `'image'` -> Return `{ action: 'handoff', reason: 'Cliente envio una imagen' }` immediately.

4. `'video'` -> Return `{ action: 'handoff', reason: 'Cliente envio un video' }` immediately.

5. `'sticker'` -> Call `handleSticker(input)`:
   - If no mediaUrl, return `{ action: 'ignore' }` (can't interpret without image)
   - Call `interpretSticker(input.mediaUrl)`
   - If gesto is not null (recognized), return `{ action: 'passthrough', text: result.gesto }`
   - If gesto is null, return `{ action: 'ignore' }` (CONTEXT.md: unrecognizable stickers are silently ignored)
   - Log interpretation result

6. `'reaction'` -> Call `handleReaction(input)`:
   - Import `mapReaction` and `reactionToMediaGateResult` from `./reaction-mapper`
   - Call `mapReaction(input.messageContent)` -- messageContent contains the raw emoji string (set by webhook handler)
   - Return `reactionToMediaGateResult(action)`

7. `default` -> Return `{ action: 'ignore' }` (unknown message type, safe to ignore)

Import createModuleLogger from '@/lib/audit/logger' for structured logging. Log at INFO level for handoffs and at DEBUG level for passthroughs.
