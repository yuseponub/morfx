  <behavior>
    Pure function: charCount (number) -> delay in milliseconds (number).

    Logarithmic curve with constants:
    - MIN_DELAY_MS = 2000 (minimum 2 seconds, even for 1 char)
    - MAX_DELAY_MS = 12000 (cap at 12 seconds)
    - CHAR_CAP = 250 (characters at which max delay is reached)
    - K = 30 (curve shape parameter -- lower = more aggressive logarithm)

    Formula: delay = MIN + (MAX - MIN) * ln(1 + chars/K) / ln(1 + CAP/K)

    Expected behavior (tolerance +/- 500ms):
    - charCount <= 0  -> 2000ms (minimum)
    - charCount = 10  -> ~2000ms
    - charCount = 50  -> ~3500ms
    - charCount = 80  -> ~5000ms
    - charCount = 100 -> ~6000ms
    - charCount = 150 -> ~8000ms
    - charCount = 200 -> ~10000ms
    - charCount = 250 -> ~12000ms (cap)
    - charCount = 500 -> ~12000ms (beyond cap, stays at max)

    Edge cases:
    - charCount = -1  -> 2000ms (treats as 0)
    - charCount = NaN -> 2000ms (treats as 0)

    Return value is Math.round() (integer milliseconds).

    Constants are exported for reference but K should NOT be configurable at runtime (adjust in code if needed after production observation).
