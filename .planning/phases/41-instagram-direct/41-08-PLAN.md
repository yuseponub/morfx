<objective>
Fix "Conectar Instagram" so it actually works. The verified root cause (proven via `scripts/_debug-ig-link.ts` against the REAL stored Varixcenter Page token): the stored Page token was minted in an early Phase 40 smoke whose `FB_LOGIN_SCOPE` deliberately excluded the IG scopes, so it can't even read `/me` (`(#100) requires 'pages_read_engagement'`) and `resolveInstagramAccount` fails even when the IG account IS professional+linked. App-level advanced-access approval is NOT the same as those scopes being GRANTED to a specific token at FB.login time.

The fix (D-IG-10/11/12): give the "Conectar Instagram" button its OWN `FB.login` popup requesting the IG superset scope (the 5 FB scopes + `instagram_basic` + `instagram_manage_messages`) with `auth_type:'rerequest'`, capture the short-lived user token, and pass it to a modified `connectInstagramAccount({ accessToken })` that runs the Phase 40 token chain verbatim, UPDATEs the facebook row's `access_token_encrypted` with the fresh superset Page token (additive — Messenger keeps working), then runs the existing `resolveInstagramAccount` + IG upsert + subscribe (now with a properly-scoped token).

Purpose: Completes the IG-03 connect path so the 41-07 cutover live smoke can actually resolve `instagram_business_account` and receive IG DMs (IG-01 / A2 linchpin). ~90% reuse — the only genuinely-new code is the FB.login popup on the IG button + prepending the 3-step token-refresh to `connectInstagramAccount`.

Output: A working IG connect button (its own login, IG-scoped token refresh) with `connect-facebook.tsx` + `connectFacebookPage` byte-identical (Regla 6 / D-IG-11).
